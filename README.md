# Time to Blyat™

**A Self-Study in Developer Rage**

> *How many minutes until you start swearing at your AI assistant? The answer may surprise you. It probably won't.*

**[Live Demo](https://evgenyq.github.io/time-to-blyat/)**

## What Is This

Time to Blyat analyzes your Claude Code session history for frustration patterns and generates a beautiful interactive dashboard. It reads your JSONL session logs, classifies emotional intensity with LLM judgment, computes metrics, and produces a single-page HTML report that turns your worst coding moments into a darkly hilarious analytics experience.

Think GitHub contribution graph, but for rage. Think NASA mission control, but for tracking how fast you go from "let me try this" to "НУ ПОЧЕМУУУУУ????".

## The Dashboard

A dark-themed scrollable page with:

- **Hero metric** — animated counter showing your median time-to-first-profanity
- **Overview stats** — Zen mode vs Rage mode ratio, carry-over trauma count
- **Frustration Curve** — how fast does the rage arrive? (with labels like "Speed Run" and "Slow Burn")
- **The Richter Scale of Rage** — your expressions classified L1 (mild "блин") through L5 (nuclear meltdown), with deadpan English translations
- **Root Cause Analysis** — what sets you off (spoiler: it's the AI ignoring your instructions)
- **Hall of Fame** — your top 3 most memorable rage quotes, presented like achievements
- **Calendar Heatmap** — a month of pain, day by day
- **By Project** — which repos hurt the most

## Prerequisites

- **Claude Code CLI** with session history in `~/.claude/`
- The **`/present`** skill available in your Claude Code (built-in skill for generating HTML presentations)

## Installation

```bash
cp -r skill/ ~/.claude/skills/time-to-blyat/
```

That's it. Run `/time-to-blyat` in Claude Code.

It will ask you about the time period, your preferred swearing language(s), and any projects to exclude. Then it does its thing with subagents and hands you a dashboard.

## What It Analyzes

- Time from session start to first frustrated message
- Intensity classification (5-level scale, from "блин" to keyboard-smashing)
- Trigger categories (build failures, AI losing context, permission prompts, etc.)
- Expression frequency and patterns
- Per-project and per-day frustration density
- Your personal escalation pattern (The Pattern™)

## How It Works

1. **Discover** — finds your Claude Code session logs in `~/.claude/projects/`
2. **Analyze** — subagents read sessions in parallel, classify frustration using LLM judgment (context matters — "блин" about pancakes ≠ "блин" after 2 hours of failing tests)
3. **Aggregate** — computes medians, distributions, rankings, selects the greatest hits
4. **Generate** — builds an interactive HTML dashboard with scroll animations, animated charts, and a film-grain overlay for that premium existential dread aesthetic

## Disclaimer

Generated by the very AI that caused most of these sessions. Your frustration patterns are stored locally and judged silently. No keyboards were harmed in the making of this report. Probably.

The data is real. The quotes are verbatim. The only fiction is the part where we pretend this is a normal thing to build.
