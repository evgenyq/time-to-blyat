# Time to Blyat™ — Developer Rage Analytics

Analyze Claude Code session history to measure how quickly the user gets frustrated, what triggers it, and present the findings as a stunning, darkly hilarious interactive dashboard.

## The Vibe

This is a self-roast, not a report. The user is voluntarily pointing a microscope at their worst moments and laughing about it. The humor comes from three sources — all three are mandatory:

1. **Authentic data** — real quotes are funnier than anything you can invent. "НУ ПОЧЕМУУУУУУУУУ????" with 40 У's is comedy gold *because it actually happened*. Never sanitize, never paraphrase. The raw emotion IS the punchline.

2. **Absurd-serious framing** — treat rage data with the gravitas of a NASA mission control dashboard. "The Richter Scale of Rage." "Carry-over trauma." "Speed Run" for sessions where frustration hit in under 10 minutes. The contrast between formal analytics language and "блять чувак, почему ты такой тупой стал?" is what makes it land.

3. **Meta-irony** — the AI that *caused* the frustration is now calmly analyzing it with charts and percentages. Lean into this. The footer disclaimer must acknowledge it: "Generated by the very AI that caused X% of these sessions. No keyboards were harmed. Probably."

### Humor Toolkit — Use These Techniques

- **Give brackets personality:** "0–10 min" → labeled "⚡ Speed Run", "60–90 min" → "🐌 Slow Burn", "already angry at start" → "😤 Carry-over Trauma"
- **Translate rage expressions with deadpan understatement:** "блин" → "Hmm, that's not ideal" / "блять" → "Nuclear option activated" / "НУ ПОЧЕМУУУ" → "🔥 Keyboard in danger 🔥"
- **Hall of Fame, not Wall of Shame:** The top 3 quotes are presented like achievements, with 🥇🥈🥉 and metadata like "Caps lock: yes • У count: 40+" or "Language barrier: transcended"
- **The Pattern™** in the footer: summarize their personal escalation sequence as a flowchart. Example: "Work calmly for 25 min → something breaks → 'блин' → 5 min later → 'бля' → 5 more min → full мат → repeat."
- **Calm sessions are "Zen mode"**, frustrated sessions are "Rage mode" — the framing is like a character stat sheet

### What NOT to Do

- Don't be mean-spirited. The user is the comedian here, you're the straight-man presenting absurd data with a poker face.
- Don't censor or soften the profanity. The whole point is honest self-analysis. "Бля" is the data, not a typo.
- Don't add disclaimers about mental health or stress management. This is comedy, not therapy.
- Don't invent fake data or embellish real quotes. The authenticity is the entire value prop.

## What This Skill Produces

A single-page scrollable interactive HTML dashboard ("1-pager", not slides) with these sections:

1. **Hero** — headline metric: median time from session start to first frustrated message, animating counter
2. **Overview Stats** — total sessions analyzed, % Zen vs Rage mode, carry-over trauma count
3. **Frustration Curve** — "How fast does the rage arrive?" — horizontal bar chart with time buckets and personality labels
4. **The Richter Scale of Rage** — taxonomy of expressions by intensity (L1 mild → L5 nuclear), with counts, original expressions, deadpan English translations, and dot indicators
5. **Root Cause Analysis — "What Sets Me Off"** — trigger categories ranked by frequency
6. **Hall of Fame — "Greatest Hits 💀"** — top 3 most memorable rage quotes with context, dates, and fun metadata
7. **Calendar Heatmap — "{Month} — Heatmap of Pain"** — day-by-day view colored by frustration density
8. **By Project — "Where The Pain Lives"** — which projects cause the most pain
9. **Footer: The Pattern™** — the user's personal escalation flowchart + self-aware disclaimer

## Workflow

### Phase 1: Discover Session Logs

Find the user's Claude Code session logs:

```
~/.claude/projects/*/         — project-level session JSONL files
~/.claude/projects/*/*.jsonl  — main session files
~/.claude/history.jsonl       — global history with timestamps and project names
```

Each `.jsonl` file is a session. Lines contain JSON objects with `type` (e.g., `"human"`, `"assistant"`), `message` content, and timestamps.

Ask the user:
- **Time period** to analyze (default: last 30 days)
- **Language context** — what language(s) do they swear in? (default: detect from logs, but ask)
- **Any projects to exclude?**

### Phase 2: Analyze Sessions with Subagents

This is compute-heavy. Use subagents to parallelize. Split sessions into ~5 chunks of ~20 sessions each.

**For each session chunk, the subagent should:**

1. Read the JSONL, extract only `"human"` messages (user's messages) with timestamps
2. Identify session boundaries — a gap of >30 minutes between messages = new logical session
3. For each logical session, classify each user message for frustration signals using LLM judgment (not just keywords!):
   - **Profanity** (any language) — explicit swear words
   - **Aggression markers** — ALL CAPS, excessive punctuation (!!!, ???), elongated words ("ПОЧЕМУУУУУ")
   - **Frustration phrases** — "опять", "я же сказал", "ну серьёзно", "are you serious", etc.
   - **Disappointment/contempt** — "разочарован", "тупой", "не понимаешь", "are you kidding", etc.
4. For frustrated sessions, record:
   - Time from session start to first frustrated message (minutes)
   - Peak intensity level (1-5)
   - Trigger category (build failure? AI ignoring instructions? permission prompts? lost context?)
   - **Best quote — the most expressive/funny frustrated message, VERBATIM, char-for-char including all extra letters and punctuation**
   - Project name (from the directory path)
   - Date

**Intensity scale:**
- L1: Mild annoyance ("блин", "hmm", sigh-like messages)
- L2: Growing frustration ("опять", "ну это же...", "did you even read...")
- L3: Real anger ("бля", "чувак", "seriously?!", cursing starts)
- L4: Full rage ("какого хера", "блять", "what the f...", "пиздец")
- L5: Nuclear meltdown (ALL CAPS, keyboard-smashing, multilingual profanity storms, sentences like "НУ ПОЧЕМУУУУУ????")

**Critical:** Use LLM judgment for classification — context matters. "Блин" as a food discussion ≠ "блин" after 2 hours of failing tests. Read surrounding messages to understand triggers.

Each subagent returns a structured summary, NOT raw logs. Keep main context clean.

### Phase 3: Aggregate Metrics

Collect all session summaries and compute:

- Total sessions analyzed
- % frustrated ("Rage mode") vs calm ("Zen mode")
- Carry-over trauma count (sessions that started already frustrated — first message within 2 min is already angry)
- Median time-to-first-frustration
- Distribution of time-to-frustration (buckets: 0-10, 10-20, 20-30, 30-45, 45-60, 60-90, 90+, "at start")
- Expression frequency table by intensity level — the raw words/phrases, how many times each appeared
- Trigger categories ranked by count
- Top 3 quotes — select for a mix of: (1) peak intensity, (2) funniest/most relatable, (3) most creative profanity. Include date, project, and colorful context for each.
- Calendar data (frustrated sessions per day, worst day highlighted)
- Per-project breakdown (which repo hurts the most)
- The user's personal escalation pattern — what's their typical trajectory from calm to angry?

### Phase 4: Generate the Dashboard

Build a single self-contained HTML file (1-pager, scrollable, NOT a slide deck). Use `/present` design sensibility but this is a dashboard, not slides.

**Design direction:**
- Dark tech aesthetic: background #0a0a0f, cards #14141f, borders #2a2a3a
- Rage palette: green (calm/L1) → yellow (L2) → orange (L3) → red (L4) → pink (L5)
- Fonts: JetBrains Mono for data/numbers, Inter for body text (Google Fonts)
- Film-grain SVG overlay at opacity 0.03 for texture
- Scroll-reveal animations (IntersectionObserver)
- Bar charts animate width on scroll with cubic-bezier easing
- Hero counter animates 0 → value with easeOutCubic
- Calendar = GitHub contribution graph style but for rage (hover tooltips)
- Quote cards with colored left border, hover → slide right

**Tone in code:** Section titles use the numbering format "01 — Overview", "02 — Timing", etc. Subtitles are the funny part: "The Numbers Don't Lie", "The Frustration Curve", "The Richter Scale of Rage", "What Sets Me Off", "Greatest Hits 💀", "Heatmap of Pain", "Where The Pain Lives".

### Phase 5: Save and Offer Deploy

Save the HTML file and offer to:
1. Open it locally in the browser
2. Deploy to GitHub Pages (create repo `time-to-blyat`, push, enable pages)
3. Just keep it as a local file

## Notes

- Session logs can be large — ALWAYS use subagents, never load full logs into main context
- The user's messages may be in any language — detect and adapt
- Privacy: if the user plans to share publicly, suggest redacting project names or sensitive quotes. But by default, include everything — they're generating this for themselves
- The dashboard title should adapt to the user's primary swear language. "Time to Blyat" is the Russian default. For English-primary users it might be "Time to F*ck" or similar. Ask if unclear.
- If the user has very few frustrated sessions (under 5), the dashboard still works — frame it as "impressively zen" and find whatever micro-frustrations exist. Even a single "hmm" can be dramatized.
